🛠️ ИСПОЛЬЗУЕМЫЕ ТЕХНОЛОГИИ
═════════════════════════════════════════════════════════════════════════════

BACKEND:
  • Node.js + Express (REST API)
  • Socket.io (WebSockets)
  • MongoDB + Mongoose (база данных)
  • Redis (кэш, опционально)
  • JWT + bcryptjs (аутентификация)

FRONTEND:
  • React 18 + Vite (быстрая сборка)
  • TailwindCSS (стили)
  • Leaflet + react-leaflet (карты)
  • Socket.io-client (real-time)
  • Zustand (state management)
  • Axios (HTTP)
  • React Router (навигация)

📝 API ENDPOINTS
═════════════════════════════════════════════════════════════════════════════

AUTH:
  POST   /api/auth/register      { name, email, password, confirmPassword }
  POST   /api/auth/login         { email, password }
  POST   /api/auth/logout

FRIENDS:
  GET    /api/friends            ← список с локациями
  GET    /api/friends/requests   ← входящие запросы
  POST   /api/friends/invite     { code }
  PUT    /api/friends/:id/accept
  PUT    /api/friends/:id/reject
  DELETE /api/friends/:id

PROFILE:
  GET    /api/profile
  PUT    /api/profile            { name, color, ghostMode, privacyMode }

NOTIFICATIONS:
  GET    /api/notifications      ← с группировкой по датам
  PUT    /api/notifications/read-all

🆘 РЕШЕНИЕ ПРОБЛЕМ
═════════════════════════════════════════════════════════════════════════════

❌ Геолокация не работает?
   → Используй localhost (не IP)
   → Разреши доступ когда браузер запросит
   → Используй приватный режим если не работает

❌ MongoDB не подключается?
   → Запусти: mongod
   → Или используй MongoDB Atlas

❌ Socket.io не подключается?
   → Проверь VITE_SOCKET_URL в client/.env
   → Проверь CORS в server/index.js

❌ Port 5000/5173 занят?
   → Отредактируй .env файлы
   → Или закрой другие приложения
